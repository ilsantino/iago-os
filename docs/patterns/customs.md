## Purpose

Provide patterns for customs and trade compliance — tariff classification, duty
calculation, export controls, and restricted party screening — using DynamoDB
for compliance records and Lambda for processing.

## Steps

### 1. DynamoDB single-table design

| Access Pattern | PK | SK | Notes |
|---------------|----|----|-------|
| Get HTS classification | `HTS#{code}` | `DETAIL` | Tariff details + duty rate |
| Get shipment customs data | `SHIPMENT#{id}` | `CUSTOMS` | Declaration, status |
| List pending declarations | GSI1: `STATUS#pending` | `SHIPMENT#{id}` | Processing queue |
| Get party screening result | `PARTY#{name-hash}` | `SCREEN#{date}` | Denied party check |
| Get trade agreement | `AGREEMENT#{code}` | `DETAIL` | FTA, preferential rates |
| List compliance docs | `SHIPMENT#{id}` | `DOC#{type}#{date}` | Commercial invoice, packing list |

### 2. Compliance workflows

**Tariff classification:**
- Lambda receives product description + origin/destination
- Looks up HTS code from reference table
- Calculates duty rate based on trade agreements
- Returns classification + estimated duties

**Restricted party screening:**
- Lambda screens buyer/seller against denied party lists
- Cache results in DynamoDB with TTL (24h — lists update daily)
- Flag matches for human review — never auto-approve

**Export control:**
- Classify items against export control lists (EAR, ITAR)
- Check destination country restrictions
- Generate required documentation

### 3. Documentation generation

| Document | Purpose | Generated By |
|----------|---------|-------------|
| Commercial invoice | Shipment value declaration | Lambda |
| Packing list | Contents manifest | Lambda |
| Certificate of origin | Origin verification | Lambda + human sign-off |
| Export declaration | Export compliance | Lambda |

Store generated documents: `pk: SHIPMENT#{id}`, `sk: DOC#{type}#{date}`.

### 4. Audit trail

Every compliance decision logged:
- `pk: AUDIT#CUSTOMS#{YYYY-MM}`, `sk: {timestamp}#{shipment-id}`
- Decision, basis (HTS code, trade agreement), reviewer, outcome
- Retention: minimum 5 years for customs records

## Output

Advisory — provides data models, workflow patterns, and compliance requirements.

## Boundaries

- Advisory patterns only — does not create infrastructure
- Does not replace licensed customs brokers — flags where human review is required
- Does not integrate with specific customs systems (ACE, CHIEF) — provides patterns
- DynamoDB single-table design only
- Does not dispatch agents
