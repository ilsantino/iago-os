## Process

Each phase of v2 daemon build runs through the standard iaGO pipeline (`scripts/execute-pipeline.sh`) via `/iago-execute`:

1. **Plan** — Write phase plans to `.iago/plans/feature-v2-<phase-slug>/NN.md` via `/iago-plan --feature`
2. **Stress test** — Pipeline step 0 (skipped if plan already contains `## Stress Test` section)
3. **Implement** — Pipeline step 1; writes code into the appropriate `runtime/<subdir>/` location
4. **Build gate** — Pipeline step 2; `tsc --noEmit && vite build` (Node side); pytest (Python side)
5. **Review** — Pipeline step 3; plan compliance + domain routing + adversarial (3-pass)
6. **Codex adversarial** — Pipeline step 4; cross-model auth/data-loss/races/rollback review
7. **Codex fix** — Pipeline step 4b; resolves all Codex findings before PR
8. **PR + tag @claude** — Pipeline steps 5–5b
9. **Summary** — Pipeline step 6; writes `.iago/summaries/<plan>-<timestamp>.md`

The `runtime/` directory grows by phase. Phase 1 adds `runtime/daemon/`, `runtime/agent-runtime/` (with `pty/` Shape 1), `runtime/telegram/`. Phase 3 expands `runtime/agent-runtime/` to add `http/`, `mcp/`. Phase 6 adds `runtime/dashboard/` (Next.js, full port). Phase 9 adds `runtime/agent-runtime/event/`. Phase 11 adds `runtime/agent-runtime/daemon/`.

**Decided constraints (do not re-litigate during planning or execution):**

- HTTP-shape adapter auth: systemd `LoadCredential=` (with strict unit sandboxing). 1Password CLI provisioning-only, never at runtime. Cross-model confirmed 2026-05-15.
- Streamlit dashboard fallback: DROPPED. Phase 6 ships the full cortextOS Next.js port directly.
- Telegram bot strategy: one bot, per-agent file-bus tagging.
- Hermes runtime: adopted in Phase 3 as Shape 3 (MCP-as-agent) via `hermes-mcp` adapter.
- AgentRuntime interface versioning: `interfaceVersion: "v1"`; RuntimeAdapterShim covers v1↔v2 migration sprints.
- O_EXCL atomic-rename for resolved output (cortextOS extension).
- session.jsonl two-phase replay (HWM-based; live intake paused during replay).

## Phase 1 acceptance criteria

Per master prompt § Acceptance criteria. Phase 1 PR must include:

1. `tsc --noEmit && vite build` exit 0 (Node), pytest exit 0 if any Python lands
2. ≥80% line coverage on `runtime/agent-runtime/` and `runtime/daemon/` new code
3. End-to-end integration test: register Claude PTY agent → claim task → Telegram approval → resume — runnable via single command
4. Inline + dedicated `runtime/<component>/README.md` per component with purpose, dependencies, configuration, ops runbook, failure modes
5. NDJSON telemetry events per phase stage, keyed on `CLAUDE_CODE_SESSION_ID`
6. Documented + tested rollback path
7. Pipeline-verified (full 8-stage `/iago-execute`), not manual git commit
8. Screenshot or terminal log evidence in PR description
