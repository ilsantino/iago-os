# `.iago/learnings/` writer contract

Source-of-truth contract for `scripts/lib/learnings-writer.sh`. Dotfile
intentionally — keeps the contract out of the consumable learnings catalog.

## Why this exists

The previous write path was a bare `printf ... >> patterns.md`. When the
parent dir was chmod 0500, when the disk was full, or when the file was
locked, the write failed silently — reviewers thought learnings were
captured while the file was untouched. The Garry-impressed completeness
standard requires loud failure. This writer enforces that.

## Function signature

```
learnings_write <pattern-key> <markdown-body>
```

Both args required. The body is appended to `$PROJECT_DIR/.iago/learnings/patterns.md`
under a `## <ISO ts> — <key>` header.

## Return codes

| Code | Meaning |
|------|---------|
| 0    | Primary write succeeded, OR fallback write succeeded (mode=fallback) |
| 1    | Write failed in fail-loud mode; OR both primary + fallback failed |
| 64   | Usage error (missing args) |

## Env-var matrix

| Var                     | Default                         | Effect |
|-------------------------|---------------------------------|--------|
| `PROJECT_DIR`           | `.`                             | Derives `$PROJECT_DIR/.iago/learnings/patterns.md` |
| `LEARNINGS_WRITE_MODE`  | `fail-loud`                     | `fail-loud` → return 1 on failure; `fallback` → write to `LEARNINGS_FALLBACK_DIR` |
| `LEARNINGS_FALLBACK_DIR`| `$PROJECT_DIR/.iago/logs`       | Where fallback writes land. Files: `learnings-fallback-YYYYMMDD-HHMMSS-$$.md`. Gitignored. |
| `CLAUDE_CODE_SESSION_ID`| (unset)                         | Threaded into telemetry events as `sessionId` field. |
| `RUN_FILE`              | (unset)                         | If set + writable, telemetry events emit to this NDJSON file. |

## Telemetry events

When `RUN_FILE` is present and writable, one of three NDJSON records is appended:

```jsonc
{"type":"learnings_written","key":"...","path":"<target>","mode":"fail-loud","err":"","ts":"<iso>","sessionId":"..."}
{"type":"learnings_written_to_fallback","key":"...","path":"<fb_path>","mode":"fallback","err":"<primary err>","ts":"<iso>","sessionId":"..."}
{"type":"learnings_write_failed","key":"...","path":"<target>","mode":"<mode>","err":"<err>","ts":"<iso>","sessionId":"..."}
```

All string fields JSON-escape literal `"` defensively (UUID-shaped session
ids never contain `\n`/`\t` in practice).

## Test coverage

`scripts/lib/learnings-writer.test.sh` covers: happy path, missing args,
permission-denied (fail-loud + fallback), parent-dir-missing recovery,
advisory-lock no-fail behavior, telemetry sessionId propagation, env-mode
overrides, disk-full simulation (best-effort skip on Windows).
