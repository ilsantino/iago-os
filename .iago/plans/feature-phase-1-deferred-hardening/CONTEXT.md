## Decided constraints (do not relitigate during planning)

These bindings are locked by the merge-train rationale (Criticals fixed in-place, all other severities deferred to one consolidated sweep) and by Phase 1 acceptance criteria #1–#8. Any plan contradicting them is wrong.

- **Scope is exactly the brief's 5-item canonical list.** Plan stack does NOT introduce new features, refactor unrelated code, or expand scope to Phase 2 work.
- **No Critical re-litigation.** Every Critical (Cn) from adv-pr41-…-46 has already landed in-place per merge train policy. This sweep handles Important + Minor + Forward only.
- **Coverage floor is 80% lines.** main.ts 62.89% → ≥80%; bot.ts 70.22% → ≥80%. Branch coverage floor is 75%. Per-file via Vitest v8 v2 `thresholds` block (already in place).
- **state-paths atomicRename API stays back-compat.** The audit MAY add a new variant (`atomicRenameStaleDest()` or `atomicRenameRace()`) but MUST NOT silently change `atomicRename()` semantics — every existing caller stays on the function they already import unless explicitly reclassified.
- **PR #45 approval-bus raw `fsp.rename` workaround stays until audit completes.** Plan 02 either keeps the raw-rename workaround with an updated JSDoc explaining the classification, OR migrates approval-bus to the new variant. Either path is acceptable — the verdict is captured in Plan 02 stress notes, not pre-decided.
- **No new MCP servers, no new daemons, no Phase 2 work.** This sweep ships entirely inside `runtime/` and `.iago/`.
- **Tests-before-fix discipline (TDD).** Every Important / Minor / Forward fix lands with a regression test that fails without the fix, passes with it. Coverage-pass tests in Plan 03 are exercise-the-uncovered-branch tests, not synthetic-line-bumpers.
- **One PR per plan, wave-grouped.** Wave 1 = plans 01 + 04 (file-disjoint). Wave 2 = plan 02 (touches state-paths.ts which 01 also imports). Wave 3 = plans 03 + 05 (file-disjoint).
- **Garry-impressed completeness standard.** Every fix idempotent; every test exercises a failure path; zero TODOs without tracked issue + date; every Forward item either lands a fix OR a tracked issue with date + link in `runtime/<component>/README.md` "Failure modes" or "Known limitations" section.
- **No surface-area expansion beyond what the reviews flagged.** New IPC features, new Telegram commands, new state-kinds, new telemetry kinds — all out of scope.
- **Pipeline preserved.** `scripts/execute-pipeline.sh` runs unchanged.

## Open questions to capture (not block) during planning

- **OQ1 — IPC `MAX_LINE_BYTES` value.** PR #44 Important #1 suggests `64 * 1024`. Default: 64 KiB unless Plan 01 finds an existing IPC payload near or above that bound.
- **OQ2 — IPC idle timeout value.** PR #44 Important #3 suggests `5 * 60 * 1000` (5 min). Default: 5 min; document the Phase 6 dashboard long-poll override path.
- **OQ3 — atomicRename audit: introduce a new variant or thread a flag?** Plan 02 picks based on caller-count + naming readability. Default: introduce `atomicRenameStaleDest()` (the destructive retry path) and keep `atomicRename` as the safer "fail on EEXIST" semantics — approval-bus PUBLISH (a) is stale-dest (resolved file is new + final); session-log HWM is stale-dest (old HWM is stale); file-bus resolved output is stale-dest. Most callers move to the new variant; few stay on the safer name.
- **OQ4 — main.ts coverage strategy.** Pure unit tests via `vi.mock` for filesystem + child_process? Or extend the `runtime/integration/hello-world.test.ts` suite? Default: hybrid — unit-test `loadPersistedConfigs` (highest uncovered branch surface) + `injectIntoAgent` + `getShapeForAgent` + `findHandleForAgent` + `resolveSessionId` + `withTimeout` + `isDirectlyExecuted` in `runtime/daemon/main.test.ts`; extend integration suite for SIGINT/SIGTERM handler paths + post-spawn shutdown guard (already partially covered, lift over the floor).
- **OQ5 — bot.ts coverage strategy.** Default: extend `runtime/telegram/bot.test.ts` to add tests for the uncovered chat-allowlist, group-chat rejection (PR #45 C1 fix), `/inject` control-byte sanitization (PR #45 C2 fix), empty-allowlist fail-loud (PR #45 C3 fix), and the 12 Important items where the fix is in place but tests are missing.
- **OQ6 — exactOptionalPropertyTypes adoption.** Plan 04 produces a decision doc (`adopt` / `defer` / `reject`) with rationale, not a code change. Default: defer to Phase 3 (where Shape 2 HTTP adapter introduces optional auth fields and the discipline pays off more).
- **OQ7 — `runtime/biome.json` configuration.** Default: mirror repo-root `biome.json` but scoped to `runtime/**` so the sub-project format hook (per memory: `feedback_subproject_format_hook.md`) stops drifting tabs↔spaces. Confirm format runs cleanly post-add.
- **OQ8 — InterfaceVersion centralization scope.** Default: extract `"v1"` literal to `INTERFACE_VERSION` const in `runtime/agent-runtime/types.ts`; replace every `"v1"` literal across `runtime/agent-runtime/` + `runtime/daemon/`. Tests must reference the const, not the literal.
