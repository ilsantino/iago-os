## Decided constraints (do not relitigate during planning)
- **File-bus is the transport.** No broker, no pub/sub, no socket bus, no ACP. The envelope rides `tasks/pending/<toAgentId>__<uuid>.json` exactly as cron does.
- **Status is the directory** (pending/claimed/resolved) — never a field in the envelope.
- **Server stamps `from`** — an agent cannot spoof the originator.
- **Chief-only producer-capability** — `role:"chief"` gates peer-task production; workers are consumers only.
- **Approval gate mandatory** for spend/prod-touching sends (reuse approval-bus).
- **Per-agent inbound quota** — resolves Open-Q11; prevents queue flooding.
- **No new AgentRuntime shape.** Chief is a config role on an existing shape.
- **No Postgres, no Docker.** Pipeline preserved.
- **Garry-impressed completeness** — every failure path tested; no TODOs without a tracked issue.

## Open questions to capture (not block)
- Free-text agent chat vs structured task/status as the primary mode (recommend structured; free-text body as escape hatch).
- Loop-depth cap via `parentTaskId` for A→B→A delegation chains.
