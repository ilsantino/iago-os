Plan output is 5–8 numbered plan files in this folder. Each plan is 2–8 tasks scoped tightly enough to stress-test independently.
2. **Stress test** — every plan stress-tested before execution. Either pass `--stress` to `/iago-plan` (inline) or run `/iago-stress` against the generated stack. Stress test surfaces precision gaps, missing acceptance criteria, contradictions with the decided constraints in the kickoff brief.
3. **Approval gate** — present the plan stack to Santiago. Do NOT skip. Do NOT proceed to execute without explicit "go".
4. **Execute** — `/iago-execute feature-v2-phase-1-daemon`. Full 8-stage pipeline per plan: stress (skipped if `## Stress Test` already in plan) → implement → build gate → review (3-pass) → Codex adversarial → Codex fix → PR → tag @claude → summary. Wave grouping permitted where dependencies allow parallel dispatch.
5. **Verify** — Phase 1 acceptance criteria 1–8 from `runtime/CONTEXT.md` § Phase 1 acceptance criteria. Self-evidence (criterion #8) means terminal log or screenshot of the hello-world handshake running on Santiago's Windows box embedded in the PR description.
6. **Session digest** — write `sessions/2026-MM-DD-iago-os-v2-phase-1.md` to the Obsidian vault per `~/.claude/rules/obsidian.md`. Update STATE.md per project rule.

## Decided constraints (do not relitigate during planning)

These bindings are locked by the ADR + master prompt + memory entries. Any plan contradicting them is wrong.

- **Multi-shape mandatory** — `AgentRuntime` is the polymorphic interface; PTY is Shape 1 of 5. Adapter declares `interfaceVersion: "v1"`.
- **One Telegram bot, per-agent file-bus tagging** — per-agent bot tokens rejected. *(Reaffirmed 2026-05-29: KEPT; per-agent identity via sticky `/agent <id>` + optional forum-topics on the single bot, tokens still rejected.)* *(SUPERSEDED 2026-05-30 — Santiago LOCKED: reversed to per-agent bots for standing agents + one chief bot for ephemeral workers; affects Phase 3+, NOT Phase 1 scope; ADR `.iago/decisions/2026-05-30-per-agent-bots-and-chief-tier.md`.)*
- **systemd `LoadCredential=` for HTTP-shape auth** (Phase 3 prereq, decision locked now). 1Password CLI provisioning-only.
- **Streamlit fallback dropped** — full Next.js dashboard ships in Phase 6, not Phase 1.
- **session.jsonl two-phase replay required in Phase 1** (pause intake → replay up to HWM → resume) — not deferred.
- **Heartbeat health checks required in Phase 1** (60s probe, RSS recycling at 512 MB default, force-restart on stall) — not deferred.
- **Subagent spawn semantics required in Phase 1** (parent-child handle linkage, cost rollup, auto-shutdown on parent exit) — not deferred.
- **PTY adapter version pinning required in Phase 1** — pin Claude Code version range, golden transcripts for prompt/status parse, fail-closed on unknown parse.
- **Per-shape lifecycle semantics binding** — vision spec § Per-shape lifecycle semantics documents Shape 1 (PTY) rules; adapter docs MUST follow them.
- **Interface versioning from day one** — registry validates `interfaceVersion` at boot.
- **O_EXCL atomic-rename for file-bus** — `wx` flag on claim, `tmp` → `rename` on resolved write, owner-ID embedded, REJECT zombie writes.
- **No Docker.** systemd on VPS directly (Phase 2). Phase 1 is local-only.
- **No Postgres.** SQLite for cost ledger (Phase 8). JSON/JSONL for everything else.
- **Pipeline preserved.** `scripts/execute-pipeline.sh` runs unchanged. Phase 1 does NOT wire daemon-driven pipeline invocation (that is Phase 2+).
- **Garry-impressed completeness standard.** No TODOs without a tracked issue + date; no "good enough for now"; every failure path handled and tested.
