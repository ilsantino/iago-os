## Decided constraints (do not relitigate during planning)

These bindings are locked by the ADR + master prompt + memory entries. Any plan contradicting them is wrong.

- **Multi-shape mandatory** ‚Äî `AgentRuntime` is the polymorphic interface; PTY is Shape 1 of 5. Adapter declares `interfaceVersion: "v1"`.
- **One Telegram bot, per-agent file-bus tagging** ‚Äî per-agent bot tokens rejected.
- **systemd `LoadCredential=` for HTTP-shape auth** (Phase 3 prereq, decision locked now). 1Password CLI provisioning-only.
- **Streamlit fallback dropped** ‚Äî full Next.js dashboard ships in Phase 6, not Phase 1.
- **session.jsonl two-phase replay required in Phase 1** (pause intake ‚Üí replay up to HWM ‚Üí resume) ‚Äî not deferred.
- **Heartbeat health checks required in Phase 1** (60s probe, RSS recycling at 512 MB default, force-restart on stall) ‚Äî not deferred.
- **Subagent spawn semantics required in Phase 1** (parent-child handle linkage, cost rollup, auto-shutdown on parent exit) ‚Äî not deferred.
- **PTY adapter version pinning required in Phase 1** ‚Äî pin Claude Code version range, golden transcripts for prompt/status parse, fail-closed on unknown parse.
- **Per-shape lifecycle semantics binding** ‚Äî vision spec ¬ß Per-shape lifecycle semantics documents Shape 1 (PTY) rules; adapter docs MUST follow them.
- **Interface versioning from day one** ‚Äî registry validates `interfaceVersion` at boot.
- **O_EXCL atomic-rename for file-bus** ‚Äî `wx` flag on claim, `tmp` ‚Üí `rename` on resolved write, owner-ID embedded, REJECT zombie writes.
- **No Docker.** systemd on VPS directly (Phase 2). Phase 1 is local-only.
- **No Postgres.** SQLite for cost ledger (Phase 8). JSON/JSONL for everything else.
- **Pipeline preserved.** `scripts/execute-pipeline.sh` runs unchanged. Phase 1 does NOT wire daemon-driven pipeline invocation (that is Phase 2+).
- **Garry-impressed completeness standard.** No TODOs without a tracked issue + date; no "good enough for now"; every failure path handled and tested.
