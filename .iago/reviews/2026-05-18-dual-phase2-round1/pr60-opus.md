## Important

- **I1 — Test 9 (ephemeral-keypair real-age round-trip) cannot succeed; `REAL_PUBKEY` extraction is broken.** `archive-openclaw.test.sh:327-330` does `age-keygen -o "$priv" 2> "$pub"` then `grep '# public key:' "$pub"`. Per upstream age-keygen source (`cmd/age-keygen/keygen.go`), when `-o` is given, the **private-key file** receives the `# public key: age1...` *comment*, while **stderr** receives the line `Public key: age1...` (no `#` prefix). The test greps stderr (`$pub`) for `# public key:`, which never matches, so `REAL_PUBKEY=""` and the next assertion `[[ -n "$REAL_PUBKEY" ]]` fails the test. Net effect: the gold-standard C2 verification (real recipient correctness, not just shape) is never actually exercised. The pipeline review missed this because Windows has no bats — test 9 has never run. **Recommendation:** change line 329 to `REAL_PUBKEY=$(grep '^# public key:' "$priv" | sed 's/^# public key: //' | tr -d '[:space:]')` (read the *private key file*, which contains the `# public key:` comment), or alternatively `grep '^Public key:' "$pub" | sed 's/^Public key: //'` to read stderr with the correct pattern.

- **I2 — `step4-encrypt "ok"` NDJSON event fires BEFORE the magic-byte (4a) and bogus-identity (4b) probes run.** `archive-openclaw.sh:287` writes `"step":"step4-encrypt","status":"ok"` immediately after `age -R`, then lines 293-331 run 4a/4b and may exit with error. Telemetry consumers parsing `cutover.ndjson` will see `step4 ok → step4a error`, which is internally inconsistent — step 4 was *not* OK if the encrypted output is unverified/unrecoverable. **Recommendation:** demote line 287 to `"status":"begin-verify"` or move the `ok` event after the 4b probe succeeds. Important rather than Minor because the NDJSON log is the canonical cutover-evidence artifact per spec §10 criterion 5; a misleading record degrades incident forensics.

## Minor

- **M1 — `TIMESTAMP` resolution is per-second; two operators invoking with `--force-new-archive` within the same wall-clock second collide.** `archive-openclaw.sh:255` uses `date -u +%Y%m%d-%H%M%S`. The second invocation overwrites the first encrypted archive AND silently writes a duplicate manifest row. Real-world hit rate is low, but the flock guard exists exactly to defend against simultaneous operators. Recommendation: append `$$` or `$(openssl rand -hex 2)` to `TIMESTAMP`.
- **M2 — flock-subshell error path in step 5 emits no `step5-manifest "error"` NDJSON event.** `archive-openclaw.sh:367-375` — if `flock -w 5` times out, the subshell `exit 1` propagates via `set -e`, but only the `"begin"` event was written. Operators reading the NDJSON see "begin" with no terminator. Recommendation: write the error event before exiting the subshell, or wrap the subshell in `set +e ... rc=$? set -e` and emit the error from the parent.
- **M3 — Step 6 (early run) unconditionally overwrites `$PRUNE_SERVICE` + `$PRUNE_TIMER` when either is missing.** `archive-openclaw.sh:174-205` — the guard is `&&`, so if a partial-install state exists (service present, timer deleted, or vice versa), both files get rewritten from the embedded heredoc, clobbering any local operator edits to the surviving file. Acceptable for a deploy script but worth documenting in the header comment.
- **M4 — `ndjson_event` "kind" field duplicates the "script" field.** Every call passes `"archive-openclaw"` as kind while the printf already emits `"script":"archive-openclaw.sh"`. Wastes log bytes and confuses downstream parsers expecting `kind` to be an event class (e.g., `stage_begin`/`stage_end`). Recommendation: either pass meaningful kinds (`stage_begin`/`stage_end`/`error`) or drop the field.
- **M5 — Decryption recipe in `MANIFEST.template.md:13` and the inline heredoc at `archive-openclaw.sh:354` use `> <file>.tar.gz` rather than spec § 4's `age -d -i ... -o <file> <file>.age`.** Functionally equivalent (redirect vs `-o`), but a spec deviation. Pick one form, document the deviation, or stay with the spec.
- **M6 — `prepare_sut` uses GNU `sed -i` (no backup ext).** `archive-openclaw.test.sh:41-52` will silently fail on BSD/macOS sed (needs `sed -i ''`). Plan acknowledges Windows-only bats run; flagging for the day someone runs the suite on Sebas's Mac.
- **M7 — Test 10's `PATH="$STUBS_DIR" run bash "$SUT_COPY"` depends on `bash` being hashed in the bats parent shell.** `archive-openclaw.test.sh:357` — if PATH is restricted before bash has been resolved, `bash` itself isn't reachable from the new PATH. Works in practice because earlier tests run bash, populating the hash table, but is fragile when run in isolation.
- **M8 — Step 1's `su - "$OPENCLAW_USER" -c "systemctl --user is-active --quiet ..."` will hard-error if `user@1001.service` is not running (no XDG_RUNTIME_DIR).** Phase 0 audit confirms it IS running; on a fresh re-provisioned VPS that constraint may not hold. Not a real-world concern at cutover, but the OpenClaw-dir-absent path (line 207) already exits cleanly when the prerequisite is missing — step 1 has no analogous defence.
