## Critical

- **C1 — `run_claude_parent_stage_end_observability_test` will FAIL when executed.** `scripts/test-pipeline-helpers.sh:451` asserts `stage_end_line == *'"sessionId":""'*` (empty). But `scripts/lib/pipeline-telemetry.sh:98` now does `export CLAUDE_CODE_SESSION_ID="${CLAUDE_CODE_SESSION_ID:-claude-${RUN_ID}-${_sid_now}-${RANDOM}}"` in **parent scope**. Trace: `unset CLAUDE_CODE_SESSION_ID; pipeline_init` → parent shell now holds a synthesized `claude-*` id → `stage_end` reads it and emits `"sessionId":"claude-..."` → assertion's FAIL branch fires with `"stage_end leaked a sessionId from the subshell: ..."`. The dispatch log at `.iago/summaries/_dispatch-c-01.log:332-340` shows the codex-fix session hit the 900s tree-kill timeout and the test suite was never re-executed after the parent-scope synthesis was introduced; only the `tsc/vite` build gate ran post-fix. CI will go red on first run. Fix: either (a) update this test to assert `*'"sessionId":"claude-'*` matching the new contract, or (b) revert `pipeline_init` parent-scope synthesis. Pick one — both can't be true at once.

## Important

- **I1 — `pipeline-telemetry.sh:5-22` header docblock contradicts the function body.** Header explicitly says: "*`pipeline_init` … RUN_SESSION_ID is NOT synthesized and NOT exported*" and "*stage_end/pipeline_finalize records emitted in the PARENT after the subshell returns will carry the OUTER env value (empty string if unset)*". But `pipeline_init:85-101` synthesizes AND exports `CLAUDE_CODE_SESSION_ID` into parent scope, and consequently `stage_end:163-168` reads the synthesized value rather than empty. Result: future maintainers reading the header will be actively misled about the actual contract. This is the exact "header lies about behavior" pattern flagged as I-A on PR #50; the codex fix introduced parent-scope synthesis but never rewrote the header. Rewrite lines 5-22 to describe what the code actually does (parent-scope synthesis happens in `pipeline_init`; `run_claude` synthesis is now defensive/redundant).
- **I2 — `outer_session_id` field name is misleading when outer env was unset.** `pipeline-telemetry.sh:115-117` captures `RUN_SESSION_ID` AFTER the synthesis-export, then emits it as `"outer_session_id"` in the `pipeline_init` NDJSON record. When the orchestrator had no real outer session, the field reports the freshly-synthesized `claude-*` id — but its name promises "outer". Plan 03's projector will join on this field believing it represents the real orchestrator session, but it'll actually point at a self-generated id with no upstream meaning. `pipeline-telemetry.test.sh` Test 12 (line 1071) only covers the env-set case and therefore never trips this. Fix: capture the outer env BEFORE the synthesis-export (`RUN_SESSION_ID="${CLAUDE_CODE_SESSION_ID:-}"` first, then do the export), so the emitted `outer_session_id` is empty when actually unset. Add a test for the env-unset case asserting `"outer_session_id":""`.
- **I3 — Plan compliance drift on Task 1.** Plan Task 1 (line 35) says "Edit `pipeline_init()` to capture `RUN_SESSION_ID="${CLAUDE_CODE_SESSION_ID:-}"`" — capture only. Plan Task 3 (line 49) confines synthesis to `run_claude` and says "**Do NOT export back to the parent shell**". PR adds parent-scope synthesis-and-export inside `pipeline_init`, which neither task authorized. This is scope creep introduced by the codex-fix loop to address Codex's PR #50 finding — the right path was to amend the plan (or open a follow-up plan) rather than silently widen scope. The new behavior IS reasonable, but it lives in two places (pipeline_init AND run_claude) and creates the dead `run_claude` synthesis branch flagged in I4 below.
- **I4 — `run_claude` synthesis (execute-pipeline.sh:156-170) is now functionally dead code.** Because `pipeline_init` always sets parent `CLAUDE_CODE_SESSION_ID` before `run_claude` ever fires, the `${CLAUDE_CODE_SESSION_ID:-$_call_sid}` fallback at line 170 can never trigger in normal pipeline flow — the `:-` left-side is always populated. `_call_sid` is computed and discarded. The standalone unit test `run_claude_synthesis_fallback_test` (line 1253 of test-pipeline-helpers.sh) keeps it green only by bypassing `pipeline_init`. Decide: either delete the `run_claude` synthesis (parent-scope handles it) or document why both paths must coexist as redundant safety nets.
