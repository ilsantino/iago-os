**I-1 (Important — silent credential check failure) — systemd-creds decrypt round-trip is silent-success on decrypt failure**
File: runtime/deploy/cutover.sh:456 (the T+05 systemd-creds verification block)
Issue: The remote pipeline `systemd-creds decrypt /etc/credstore.encrypted/iago-telegram-token.cred - | wc -c` runs under ssh's sh -c (no pipefail). If `systemd-creds decrypt` fails (corrupt ciphertext, missing key, wrong owner), wc -c reads empty stdin, prints "0", exits 0. The local `if vssh ...` sees success. The whole point of the check (catch busted creds before T+07) is silently neutered.

Fix: Use the length-capture pattern (defensive):
```bash

if (( len > 1 )); then
  echo "  OK systemd-creds round-trip verified (${len} bytes)"
else
  trigger_rollback "systemd-creds decrypt round-trip empty/failed for iago-telegram-token (len=${len})"
fi
```

Apply the same pattern to any other systemd-creds round-trip check (gh-token if present).
